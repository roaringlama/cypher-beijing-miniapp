# Test Plan

## 验证层级

1. 静态规范：格式、JSON、页面清单、敏感信息扫描。
2. 类型检查：共享领域模型、Service 接口、页面状态。
3. 单元测试：报名容量/重复、地址投影、审核与取消、积分幂等、等级、Feature Flag、Mock 赞助、网络重试。
4. 构建检查：每个 app.json 页面具备 `.ts/.json/.wxml/.wxss`，数据模型 13 个集合齐全。
5. 视觉验收：390×844 视口逐页检查首页、详情、创建、个人和赞助；确认中文不溢出、主按钮可见、导航一致。

## Checkpoint A 通过条件

- 五个指定核心页面可浏览并有清晰截图；
- 首页可筛选，详情可进入报名确认，创建流程可前后切换，设置影响反馈，赞助明确为测试模式；
- `party/video/practice` 默认不可见；
- 音效/震动/减少动画开关存在，快速点击和重复提交受保护；
- `npm run verify` 全部通过；敏感信息扫描无命中。Checkpoint A 当前共有 19 项自动规则测试。

## Phase 2 以后补充

云函数契约测试、权限模拟、并发报名、支付回调签名、弱网重试、真机兼容、无障碍和性能基线。
