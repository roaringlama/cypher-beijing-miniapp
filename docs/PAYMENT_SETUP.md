# Payment Setup

## 当前状态

Phase 1 仅使用 `MockPlatformSupportPaymentProvider`。页面必须持续显示“测试模式 · 不会真实扣款”，Mock 成功不得增加舞力、等级、活动曝光或任何特权。

## 真实微信支付前置（Phase 6，硬阻塞）

需要用户提供并授权：商户号、API v3 密钥、平台证书/序列号、商户私钥、回调域名和关联 AppID。所有秘密只进入安全密钥管理或本机忽略文件，禁止 Git 提交。

## 服务端规则

- 客户端只提交金额档位、留言、匿名与上墙选择；服务端校验金额。
- 业务订单号和 `idempotencyKey` 唯一，重复请求返回同一订单。
- 最终状态只接受验签后的支付回调或服务端主动查单。
- 回调验签、金额/商户/AppID 一致性校验后再置为已支付。
- 日志屏蔽身份证、OpenID、密钥、证书和完整支付报文。
